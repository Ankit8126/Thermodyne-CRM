# Alembic Module

## Yeh module kyu bana hai?

Alembic database migrations handle karta hai — jab bhi koi SQLAlchemy model change ho (naya table, naya column, column modify), toh Alembic se migration generate aur apply hoti hai. Bina migration ke directly DB schema change mat karo.

## Structure

```
alembic/
├── env.py             # migration runner config (async SQLAlchemy)
├── script.py.mako     # migration file template
└── versions/          # generated migration files
    └── .gitkeep
```

## Files

### `env.py` — Migration Environment

- Async SQLAlchemy engine use karta hai (`create_async_engine`)
- `target_metadata = Base.metadata` — isse Alembic ko pata chalta hai ki current models kya hain
- **Important**: Jab naya model banao toh yahan import karo taaki autogenerate me aaye:
  ```python
  from app.models.user import User  # noqa: F401
  from app.models.lead import Lead  # noqa: F401  ← naya model add karo
  ```
- `run_async_migrations()` — online migration async mode me chalti hai

### `script.py.mako` — Migration Template

- Jab `alembic revision --autogenerate` run karo toh is template se naya migration file banta hai
- Isko edit mat karo unless migration format change karna ho

### `versions/` — Migration Files

- Autogenerated migration files yahan aate hain
- Har file me `upgrade()` aur `downgrade()` functions hote hain
- **Kabhi manually generated migration delete mat karo** agar already applied ho

## Common Commands (Docker ke andar)

```bash
# Naya migration generate karo
make makemigrations msg="add leads table"

# Pending migrations apply karo
make migrate

# Ya directly container me:
docker compose -f docker-compose.dev.yml exec backend uv run alembic upgrade head
docker compose -f docker-compose.dev.yml exec backend uv run alembic downgrade -1   # rollback last
docker compose -f docker-compose.dev.yml exec backend uv run alembic history         # migration history dekho
```

## Rules

- Model change ke baad hamesha autogenerate migration run karo
- Migration apply karne se pehle generated file review karo — autogenerate sab kuch detect nahi karta (e.g. data migrations manually likhni padti hain)
- Production me downgrade mat karo bina sochhe — data loss ho sakta hai
- Migration files ko git me commit karo
